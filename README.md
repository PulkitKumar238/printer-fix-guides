# PrinterFix Guides

A content-rich, statically generated printer troubleshooting site built with
Next.js 14 (App Router), TypeScript, and Tailwind CSS. Every page is prerendered
(SSG) for fast loads and clean SEO.

## Quick start

```bash
npm install
npm run dev          # http://localhost:3000  (use PORT=3210 npm run dev if 3000 is taken)
npm run build        # production build (prerenders all 30 pages)
npm run start        # serve the production build
```

## How content works

Guides, error codes, and brand hubs are **structured TypeScript data files** —
not hardcoded JSX and not loose MDX prose. This is a deliberate choice: each
guide’s rendered UI *and* its schema.org JSON-LD (HowTo + FAQPage) are generated
from the same object, so the structured data can never drift out of sync with
what the visitor reads.

```
content/
  guides/          one file per problem guide (setup, offline, drivers, …)
  errors.ts        all error-code pages (add an object to add a page)
  brands.ts        brand hub pages (HP, Canon, Epson, Brother)
lib/
  types.ts         the content model every page conforms to
  schema.ts        JSON-LD builders (HowTo, FAQPage, Breadcrumb, WebSite)
  search-index.ts  static client-side search index
components/        reusable UI (DiagnosticSteps is the signature element)
app/               routes — fixed paths for guides, dynamic for errors & brands
```

### Adding a guide

1. Create `content/guides/<slug>.ts` exporting a `Guide` object.
2. Register it in `content/guides/index.ts` (`guides` map + `guideOrder`).
3. Add a route file `app/<slug>/page.tsx` (copy any existing one — 12 lines).

### Adding an error code

Append an `ErrorCode` object to the array in `content/errors.ts`. The dynamic
route, sitemap, and brand hubs pick it up automatically.

## Images

Step illustrations are branded **SVG placeholders** under
`public/images/guides/<slug>/step-N.svg`, generated by
`node scripts/gen-placeholders.mjs`. To use real photos, drop a raster image at
the same path and update the `image.src` in the content file — no component
changes needed. `next/image` is used throughout with descriptive alt text;
below-the-fold images lazy-load.

Brand hub pages show a real product photo beside the intro, loaded by brand key
from `public/images/brands/<key>.jpg` (`hp`, `canon`, `epson`, `brother`). These
are real printer photos from Wikimedia Commons — swap any file at the same path
to change the image, no code change needed. Check the source file's license on
Commons if you need formal attribution.

The home, about, contact, and error-code list pages each carry a general printer
photo from `public/images/site/` (also real photos from Wikimedia Commons). Each
error-code detail page reuses its brand photo from `public/images/brands/`.

## SEO

- Single `<h1>` per page, logical `h2`/`h3` nesting.
- Per-page `<title>`, meta description, canonical URL, Open Graph + Twitter tags.
- `app/sitemap.ts` and `app/robots.ts` generate `/sitemap.xml` and `/robots.txt`.
- HowTo + FAQPage + BreadcrumbList JSON-LD on guide and error pages.

## Deployment (Vercel)

Set `NEXT_PUBLIC_SITE_URL` to the production origin (e.g.
`https://yourdomain.com`) so canonical URLs, OG image URLs, sitemap, and
structured data resolve to absolute URLs. Then deploy as a standard Next.js
project — no other configuration required.

## Design tokens

Defined in `tailwind.config.ts`: `ink`, `amber`, `paper`, `surface`, `slate`,
`success`. Fonts: Zilla Slab (headings), Inter (body), JetBrains Mono (codes),
loaded via `next/font`.
